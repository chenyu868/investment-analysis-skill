# 投资分析报告 Skill

这是一个用于生成投资分析报告的 Codex Skill。它适合课程作业、公司研究、行业研究和投资备忘录草稿，通过人机协作的方式完成从分析计划、资料核验、报告撰写到多轮修改的完整流程。

本仓库不会自动给出买卖建议，而是帮助使用者形成有依据、可追溯、可复核的投资价值分析。

## 适用场景

- 上市公司或非上市公司的投资价值分析
- 行业研究、主题研究、竞品比较
- 课程论文、课堂展示、研究报告初稿
- 多家公司横向比较和筛选
- 需要保留人机协作过程的报告写作

## 能做什么

- 制定投资分析计划
- 分析行业发展趋势
- 判断公司发展趋势是否匹配行业趋势
- 分析政策趋势，并判断公司是否匹配政策方向
- 输出 SWOT 分析
- 比较公司与同行的优势和劣势
- 梳理上游、下游、客户和配套产业
- 做财务质量、估值或情景分析
- 记录资料来源、假设条件和人机协作日志
- 给出平衡、可验证的投资价值结论

## 安装方式

将本仓库克隆到 Codex 的 skills 目录：

```bash
cd ~/.codex/skills
git clone git@github.com:chenyu868/investment-analysis-skill.git
```

如果你的 Codex skills 目录不是 `~/.codex/skills`，请把仓库放到你实际使用的 skills 目录下。

## 使用方式

在 Codex 中可以这样调用：

```text
Use $investment-analysis-skill to help me write an investment analysis report about 比亚迪.
```

也可以直接用中文描述需求：

```text
帮我用 investment-analysis-skill 写一份宁德时代投资价值分析报告，要求包含行业趋势、政策趋势、SWOT、同行比较、产业链和风险分析。
```

如果没有指定公司或行业，Skill 会先帮助你确认研究对象、报告用途、格式、长度、资料来源和需要人工核验的内容。

## 工作流概览

1. 确认研究对象、报告用途和交付格式。
2. 根据公司、行业或投资主题确定报告范围。
3. 区分事实、假设和判断，建立证据基础。
4. 按模板撰写报告草稿。
5. 与用户逐轮修改，记录协作日志。
6. 输出最终报告、资料来源、假设列表和待核验问题。

## 输出内容

通常会包含：

- 投资分析报告正文
- 执行摘要和投资逻辑
- 行业趋势与政策趋势分析
- 公司与趋势匹配度判断
- SWOT 分析
- 同行对比表
- 上下游产业链、客户和配套产业分析
- 财务质量、估值或情景分析
- 风险因素和反证条件
- 人机协作日志
- 资料来源和待核验清单

## 注意事项

- 当前股价、财务数据、政策文件和市场排名需要使用最新可靠来源核验。
- 优先使用公司公告、年报、交易所披露、监管机构文件和公司投资者关系页面。
- 如果资料来源冲突，应说明冲突，并优先采用官方披露。
- 报告中的估值、增长率、利润率和情景概率都应明确标注为假设。
- 不应使用“必涨”“稳赚”“无风险”等无法验证的表述。
